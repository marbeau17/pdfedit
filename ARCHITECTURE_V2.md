## 結論: 「ローカルファースト」アーキテクチャ

PDFファイルの全処理をブラウザ内で完結。サーバーにPDFを送信しない。

```
現行:  ユーザー → [PDF Upload 4.5MB制限] → Vercel → PyMuPDF → 結果返却
新規:  ユーザー → [ブラウザ内で全処理] → 完了 (サーバー通信なし)
AI時:  ユーザー → [テキスト抽出] → [テキストのみ送信] → Gemini API → 結果返却
```

## アーキテクチャ図

```
┌─────────────────────────────────────────────────────────┐
│                  ユーザーのブラウザ                        │
│                                                         │
│  FileReader API → Web Worker (PDF.js + pdf-lib) → Canvas │
│       ↕                    ↕                             │
│  IndexedDB            テキスト抽出                        │
│  (自動保存/履歴)       (AI機能時のみ)                     │
│                            │                             │
│                    テキストのみ送信                        │
│                    (PDFは送信しない)                      │
└────────────────────────────┼──────────────────────────────┘
                             │ HTTPS (JSON, 最大1MB)
                             ▼
┌─────────────────────────────────────────────────────────┐
│  Vercel (最小限のサーバー)                                 │
│  POST /api/ai/analyze   → Gemini API (テキスト処理のみ)   │
│  POST /api/ai/generate  → Gemini API                    │
│  GET  /api/health       → ヘルスチェック                  │
└─────────────────────────────────────────────────────────┘
```

## クライアント vs サーバー 責務分離

### ブラウザ内 (制限なし)
| 機能 | ライブラリ |
|------|-----------|
| PDF表示・レンダリング | PDF.js |
| ページ操作 (追加/削除/並替/回転) | pdf-lib |
| PDF結合・分割 | pdf-lib |
| テキスト・画像追加 | pdf-lib |
| 透かし除去・ブランディング | pdf-lib + Canvas |
| エリア画像置換 | Canvas API |
| 最適化 | pdf-lib |
| 一時保存・Undo | IndexedDB (Dexie.js) |
| ファイル保存 | Blob + URL.createObjectURL |

### サーバー (テキストのみ)
| 機能 | 技術 |
|------|------|
| AI解析 (Gemini プロキシ) | FastAPI API Route |
| APIキー管理 | Vercel環境変数 |
| レート制限 | ミドルウェア |

### 廃止
- PDFアップロード (/api/upload)
- PyMuPDF サーバー処理
- in-memory セッションストレージ
- Python PDFランタイム

## ファイルサイズ制限

| 項目 | 制限 |
|------|------|
| PDFファイル | **制限なし** (デバイスメモリ依存) |
| 推奨最大 | 500MB |
| 快適操作 | 100MB以下 |
| AI テキスト送信 | 1MB/リクエスト (チャンク分割で無制限) |

## 技術スタック

| カテゴリ | 技術 | 用途 |
|---------|------|------|
| PDF表示 | PDF.js (Mozilla) | レンダリング・テキスト抽出 |
| PDF編集 | pdf-lib | 構造編集・ページ操作 |
| ストレージ | IndexedDB + Dexie.js | クライアント側永続化 |
| 並列処理 | Web Workers | UIブロック防止 |
| 描画 | Canvas API | 注釈・エリア選択 |
| UI | HTMX + Alpine.js + Tailwind | フロントエンド |
| サーバー | FastAPI (最小限) | AI プロキシのみ |
| AI | Gemini API | テキスト処理 |
| デプロイ | Vercel | CDN + サーバーレス |

## メリット

- ファイルサイズ制限撤廃 (4.5MB → 無制限)
- アップロード待ち時間ゼロ
- プライバシー強化 (ファイルがサーバーに出ない)
- インフラコスト 50-65% 削減
- スケーラビリティ問題解消
